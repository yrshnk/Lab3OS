Проект представляет собой консольное многопоточное приложение на C++, демонстрирующее синхронизацию потоков с использованием примитивов WinAPI и обработку сценария «тупика». Главный поток (main) создаёт массив целых чисел заданной размерности, инициализирует его нулями и запускает N потоков marker, передавая каждому его порядковый номер.

Поток marker после сигнала hStartSignal генерирует случайные индексы и пытается пометить ячейку массива своим идентификатором, используя критическую секцию (CRITICAL_SECTION cs) для взаимного исключения. Если выбранная ячейка занята, поток выводит информацию о своей работе, сигнализирует главному потоку через hFinishEvents и ждёт события hRemoveEvent — на продолжение (hContinueSignal) или завершение. При завершении поток снимает свои пометки и корректно выходит.

Главный поток циклически ждёт сигналов от маркеров (WaitForMultipleObjects), выводит текущее состояние массива, запрашивает у пользователя поток для завершения, выставляет rem, уведомляет маркеры (PulseEvent(hRemoveEvent)), дожидается завершения выбранного потока и возобновляет работу остальных (PulseEvent(hContinueSignal)). Цикл повторяется до завершения всех маркеров.

Реализация выполнена на стандарте C++17, использует WinAPI для потоков и синхронизации (CreateThread, CreateEvent, WaitForSingleObject, WaitForMultipleObjects, InitializeCriticalSection, EnterCriticalSection, LeaveCriticalSection, DeleteCriticalSection, PulseEvent, CloseHandle, Sleep) и стандартные контейнеры и ввод/вывод. 

Тестирование с GoogleTest покрывает корректное помечание массива одним и несколькими маркерами, снятие пометок при завершении потока и повторное использование массива. Обработка ошибок включает проверку создания потоков и событий, валидацию ввода размеров и номеров потоков, а также корректное освобождение ресурсов.
